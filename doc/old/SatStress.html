SatStress.py calculates the tidal stresses on viscoelastic icy satellites.
 
This Python module is an implementation of the model described in the Icarus
paper "Modeling Stresses on Satellites due to Non-Synchronous Rotation and
Orbital Eccentricity Using Gravitational Potential Theory" by Wahr et al.
(2008).
 
It defines several objects which allow you to calculate tidal stresses on an
icy satellite.  Rather that answering a particular question, it is meant to
facilitate the creation of a variety of tools for testing various hypotheses
related to lineament formation and tidal stress fields.  It is designed to be
extended over time to include stresses from sources other than orbital
eccentricity and non-synchronous shell rotation.
 
Use of this module requires that you have the Scientific Python (SciPy) package
installed.  It is available for Windows, Linux, Unix, and Mac OS X.  For more
information on SciPy see:

class NSR(StressDef)
    A derived class (descends from StressDef) that represents the stresses that
arise on the surface of an icy satellite due to the non-synchronous rotation of
a decoupled icy shell floating atop an inviscid ocean over a synchronously
rotating core.  See detailed discussion in Wahr et al. (2008).
 
  Methods defined here:
Tpp(self, r, theta, phi, t)
Calculates the Tau_{phi phi} (east-west) component of the stress tensor.
Tpt(self, r, theta, phi, t)
Calculates the Tau_{phi theta} (off-diagonal) component of the stress tensor.
Ttt(self, r, theta, phi, t)
Calculates the Tau_{theta theta} (north-south) component of the stress tensor.
__init__(self, satellite)
Initialize the definition of the stresses due to NSR of the ice shell.
 
The forcing frequency omega is the frequency with which a point on the
surface passes through a single hemisphere, because the NSR stress field
is degree 2 (that is, it's 2x the expected omega from a full rotation)
 
Because the core is not subject to the NSR forcing (it remains tidally
locked and synchronously rotating), all stresses within it are presumed
to relax away, allowing it to deform into a tri-axial ellipsoid, with
its long axis pointing toward the parent planet.  In order to allow for
this relaxation the shear modulus (mu) of the core is set to an
artificially low value for the purpose of the Love number calculation.
This increases the magnitude of the radial deformation (and the Love
number h2) significantly.  See Wahr et al. (2008) for complete
discussion.

class SatLayer
    Class describing a homogeneous material layer within a satellite.
 
  Methods defined here:

Set the values of parameters defining the a satellite layer.
 
satParams is a dictionary constructed from name value pairs read in
from a satellite specification file.
 
layer_n indicates which layer in the satellite is being defined, with
n=0 indicating the center of the satellite, and increasing outward.
The layer by itself has no indication of where within the satellite it
resides.  That information is contained in the ordering of the list of
layers within the satellite object.
 
The layer defining parameters are:
 
* label:       string describing the layer (e.g. core)
* density:     density of the layer [kg m^-3]
* lame_mu:     real-valued Lame parameter mu (shear modulus) [Pa]
* lame_lambda: real-valued Lame parameter lambda [Pa]
* thickness:   how thick the layer is, in [m]
* viscosity:   viscosity of the layer [Pa s]
* tensile_str: tensile strength of the layer [Pa]
 
There are also derived quantities that depend on the above parameters
which are convenient to have available.  They are implemented as the
following methods:
 
* maxwell_time():    The relaxation time of the material, equal to
                     (viscosity/lame_mu). [s]
 
* bulk_modulus():    The bulk modulus (kappa) of the material. [Pa]
 
* youngs_modulus():  Young's modulus (E). [Pa]
 
* poissons_ratio():  Poisson's ratio (nu).
 
* p_wave_velocity(): Velocity of a pressure wave in the material [m s^-1]

class Satellite
    A class describing the physical structure and context of a satellite.
 
Defines a satellite's material properties, internal structure, orbital
context, and the tidal forcings to which it is subjected.
 
The Satellite is initialized from an input file consisting of name=value
pairs, one per line (as described under nvf2dict).  The file must define
the following parameters, all of which are specified in SI (MKS) units,
with the exception of NSR_PERIOD_YEARS, which is in terrestrial years.
 
         SYSTEM_NAME:  A string identifying the planetary system, e.g.
                       JupiterEuropa.
 
         PLANET_MASS:  The mass of the planet the satellite orbits [kg].
 
  ORBIT_ECCENTRICITY:  The eccentricity of the satellite's orbit.  Must not
                       exceed 0.25.
 
ORBIT_SEMIMAJOR_AXIS:  The semimajor axis of the satellite's orbit [m].
 
          NUM_LAYERS:  The number of layers within the satellite.
                       (Currently NUM_LAYERS must equal 4, because that is
                       what the Love number code expects)
 
          NSR_PERIOD:  The time it takes for the satellite's icy shell to
                       undergo one full rotation [s]
 
           LOVE_PATH:  The absolute path to the program used to calculate
                       the frequency-dependent degree-2 Love numbers.
                       Currently this code is one provided by John Wahr.
 
In addition to the above parameters, the input file must define several
materially homogeneous layers, describing the internal structure of the
satellite.  Currently, due to the way the Love numbers are calculated,
four layeres must be specified: core, ocean, ice_lower, and ice_upper.
 
Each layer is composed of seven parameter values, and each layer has a
unique numeric identifier, appended to the end of all the names of its
parameters.  Layer zero is the core, with the number increasing as the
satellite is built up toward the surface.  In the below list the "X" at the
end of the parameter names should be replaced with the number of the layer.
 
           LABEL_X:  A string identifying the layer, e.g. "ocean" or "core"
 
         DENSITY_X:  The density of the layer at zero pressure [m kg^-3]
 
         LAME_MU_X:  The real-valued Lame parameter mu (shear modulus) [Pa]
 
     LAME_LAMBDA_X:  The real-valued Lame parameter lambda [Pa]
 
       THICKNESS_X:  The thickness of the layer [m]
 
       VISCOSITY_X:  The viscosity of the layer [Pa s]
 
TENSILE_STRENGTH_X:  The tensile strength of the layer [Pa]
 
Not all parameters need necessarily be defined for all layers.  For
example, the tensile strength and viscosity of the ocean are irrelevant to
the calculation.

class StressDef
    An object that defines a stress field, depending on the satellite.
 
Different tidal forcings are specified as sub-classes of this superclass (one
for each separate forcing).  Currently only the NSR and Diurnal forcings have
been coded up (see Wahr et al. 2008 for a discussion of those forcings)
 
Time is specified in seconds, with zero occuring at periapse, in order to be
compatible with the future inclusion of stressing mechanisms which may have
explicit time dependence instead of being a function of the satellite's orbital
position (e.g. a true polar wander trajectory).
 
Location is specified within a polar coordinate system having its origin at the
satellite's center of mass, using the following three variables:
 
radius      (r):     The distance between the a point and the center of
                     mass of the satellite.  Because the model currently only
                     calculates membrane stresses at the satellite's surface,
                     this should be set to satellite.radius(), though it is
                     presently ignored in the actual calculation.
 
co-latitude (theta): The arc separating a point on the surface of the
                     satellite from the north pole (0 < theta < pi).
 
longitude   (phi):   The arc separating the meridian of a point and the
                     meridian which passes under the average location of
                     the primary (planet) in the sky over the course of an
                     orbit (0 < phi < 2 pi).  East is taken as positive.
 
Each subclass much define at the three components of the membrane stress
tensor, Ttt, Tpp, and Tpt (the north-south, east-west, and shear stress
components) as methods.

The file should contain a series of NAME=VALUE pairs, one per line.  Blank
lines are ignored, as are lines beginning with the comment character
(assumed to be the pound or hash symbol '#' unless otherwise specified).
End of line comments are also stripped.  String values should not be quoted
in the file.  Names are case sensitive.
 
Returns a Python dictionary that uses the NAMEs as keys, with the VALUEs as
values, and so all Python limitations on what can be used as a dictionary
key apply to the NAME fields.
 
If a VALUE can be converted to a float, it is, everything else is left as a
string, with leading and trailing whitespace stripped.
 
Raises exceptions if:
  - a non-comment line is found not to contain a '=' character
  - there are no alphanumeric characters on one side of the '='
  - the NAME cannot be used as a key in a dictionary.
